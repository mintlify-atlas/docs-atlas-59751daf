The Magic ObjDB data model is built around a card (`Carta`) hierarchy using JPA's **JOINED inheritance strategy**. Three concrete card types extend the abstract base, while `Mazo` (deck) and `Jugador` (player) form the ownership layer.

## JOINED inheritance strategy

`Carta` declares `@Inheritance(strategy = InheritanceType.JOINED)`. This means:

- A single **`Carta` table** holds all shared columns (`id`, `nom`, `descripcio`, `raresa`, `edicio`, and the embedded `CostMana` columns).
- Each concrete subclass gets its **own table** containing only the columns specific to that type, linked to `Carta` via a shared primary key.
- Querying any subtype performs a `JOIN` between the `Carta` table and the subtype table.

<Note>
  JOINED is the most normalized inheritance strategy. It avoids null columns but requires a join per query. For a card collection where each subtype has distinct fields, this is the appropriate choice.
</Note>

## Entity summary

| Entity | Type | Table(s) | Description |
|--------|------|----------|-------------|
| `Carta` | Abstract `@Entity` | `Carta` | Base card. Never instantiated directly. Holds shared fields and the embedded `CostMana`. |
| `Criatura` | Concrete `@Entity` | `Carta` + `Criatura` | Creature card with power/toughness and flying ability. |
| `Terra` | Concrete `@Entity` | `Carta` + `Terra` | Land card. Produces colored mana; zero mana cost to play. |
| `Encanteri` | Concrete `@Entity` | `Carta` + `Encanteri` | Spell/enchantment card with a type and instant flag. |
| `Mazo` | `@Entity` | `Mazo` | A named deck holding a list of cards via `@ManyToMany`. |
| `Jugador` | `@Entity` | `Jugador` | A player with a level, owned decks, and a card collection. |
| `CostMana` | `@Embeddable` | _(inline in `Carta`)_ | Mana cost broken down by color. Stored as columns in the `Carta` table. |

## CostMana embeddable

`CostMana` is annotated `@Embeddable` and embedded in `Carta` via `@Embedded`. It stores the mana cost as six integer columns вҖ” one per color вҖ” directly inside the `Carta` table. There is no separate `CostMana` table.

```java
// Stored as: blanc, blau, negre, vermell, verd, incolor columns in CARTA
@Embedded
private CostMana cost;
```

Each field maps to a Magic: The Gathering mana symbol:

| Field | MTG Symbol | Color |
|-------|-----------|-------|
| `blanc` | `{W}` | White |
| `blau` | `{U}` | Blue |
| `negre` | `{B}` | Black |
| `vermell` | `{R}` | Red |
| `verd` | `{G}` | Green |
| `incolor` | `{C}` | Colorless |
